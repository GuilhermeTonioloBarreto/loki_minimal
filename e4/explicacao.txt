o objetivo deste exemplo é filtrar as querys a partir do próprio texto do log utilizando o operador de filtro literal |= (contém)

- rode o arquivo ready.sh para saber se o servidor está pronto para ser utilizado
- rode o arquivo query.sh. não é para aparecer nenhum log, visto que nada foi adicionado ainda
- rode o arquivo post-eps-M1.sh. um log com label job = quati e com o texto de log relatando problemas no espelho M1 será adicionado
- rode o arquivo post-eps-M2.sh. um log com label job = quati e com o texto de log relatando problemas no espelho M2 será adicionad
- rode o arquivo query.sh. é para aparecer os dois logs de espelho

- agora, rode o arquivo query-M1.sh. veja que apenas o log de M1 apareceu. isso porque, na query, está sendo utilizado o símbolo |=, que significa que deve-se ignorar os logs que não tem a palavra "M1" no texto do log
- agora, rode o arquivo query-M2.sh. veja que apenas o log de M2 apareceu. isso porque, na query, está sendo utilizado o símbolo |=, que significa que deve-se ignorar os logs que não tem a palavra "M2" no texto do log
